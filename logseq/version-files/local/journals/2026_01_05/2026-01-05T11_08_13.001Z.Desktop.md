	- 12:54 [[IDB]] 原生方法实现单字段精确查询 #reading
	  id:: 695b437d-346f-4a0b-83e1-b0342336be7d
	  collapsed:: true
		- 原生方法实现的单字段精确查询中，需要理解“主键”和“索引”的区别。
		- 要实现该功能需要先[初始化 DB](((6956acae-2332-4bd6-9457-5ea1dc9dedd9)))
		- ```ts
		  const getTodosByStatus = (db: IDBDatabase, status: 'done' | 'pending') => {
		    return new Promise((resolve, reject) => {
		      try {
		        // 1. 开启事务
		        const tx = db.transaction(['todos'], 'readonly');
		        
		        // 2. 获取存储仓库
		        const store = tx.objectStore('todos');
		        
		        // 3. KEY: 获取索引
		        // 确保初始化 DB 时创建了这个索引
		        const idx = store.index('by_status');
		        
		        // 4. 执行查询
		        // 注意：getAll 查所有数据，大数据查询会卡死
		        const request = idx.getAll(status);
		        
		        // 5. 监听查询成功事件
		        request.onsuccess = () => {
		          console.log(`Found ${request.result.length} items.`);
		          resolve(request.result);
		        };
		        
		        // 6. 监听查询失败事件
		        request.onerror = () => {
		          reject(request.error);
		        };
		      } catch (error) {
		        reject(error);
		      }
		    });
		  };
		  ```
- [[Git]]
	- 13:39 修改**最近几次**的 commit #coding
	  collapsed:: true
		- 用 `git rebase -i HEAD~n`来修改最近 `n` 此的 commit 信息，比如修改最近 3 次的 commit 信息：
		- ```bash
		  git rebase -i HEAD~3
		  ```
		- 在打开的编辑器中对需要修改的 commit 前的 `pick` 改为 `edit` (缩写 `e`)，之后保存并关闭编辑器。
		- 对每个被标记为 `edit` 的 commit 执行 `：
		- ```bash
		  git commit --amend
		  ```
		- 如果只修改作者信息则可以：
		- ```bash
		  git commit --amend --author="Your Name <your.email@example.com>" --no-edit
		  ```
		- 编辑完 commit 信息后执行 `git rebase --continue` 继续对下一个 commit 进行修改，知道修改完所有标记为 `edit` 的 commit。
		- 该操作会重写 git history 历史，因此提交时需要添加 `--force-with-lease` 选项 (`git push --force-with-lease`)。
