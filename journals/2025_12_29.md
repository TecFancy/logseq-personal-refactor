- 13:09 学习 [[Learn IndexedDB]] 事务模型 #reading
	- IDB 是一个**事务型数据库系统**，因此，数据的增删改查操作都必须在事务作用域内执行，以确保数据的完整性和一致性。
	- IDB 有三种事务模型 (定义层)
		- readonly (只读)
			- IDB 默认便是只读模式。
		- readwrite (读写)
			- 只有处于该模式下才能对数据执行修改操作。
		- versionchange (版本变更)
			- 要创建或删除[对象存储](((69520ef2-6eb5-4685-8bde-a4d992ce498f)))、索引，必须在这个模式下进行，该模式只会在 `onupgradeneeded` 事件中触发。
	- 自动回滚：如果事务中存在任意失败的请求且未被处理，整个事务会自动回滚，确保数据准确性，保证数据不会处于“中间状态”。
- 13:17 学习 [[Learn IndexedDB]] 中的对象存储 #reading
  id:: 69520ef2-6eb5-4685-8bde-a4d992ce498f
  collapsed:: true
	- IDB 的对象存储 (Object Store) 是存放数据的容器，既能存储 JavaScript 对象，也可以存储字符串、数字等原始类型的值。
	- 其角色类似于传统关系型数据库中的“表”，但结构更灵活。
	- 以下是 IDB 对象存储的核心特性
		- 创建时机：对象存储只能在 `onupgradneeded` 事件中创建或修改。只有在数据库版本变更时才会触发该事件，也就是说，只有第一次定义数据库版本或后续更新数据库版本时会触发 `onupgradeneedes` 事件，在该事件中创建或修改对象存储。
		  logseq.order-list-type:: number
		- 面向对象：不需要事先定义列信息，可以存储任何**[结构化克隆算法](((69521317-6d76-4ecb-828f-10b9aa1a6b05)))**支持的对象，不需要像关系型数据库那样预先定义复杂的列信息。
		  logseq.order-list-type:: number
		- 键值管理：对象存储中的每条数据都通过一个唯一的“键 (Key)”来检索。
		  logseq.order-list-type:: number
			- 可以将对象中的某个属性作为**键路径 (keypath)**；
			  logseq.order-list-type:: number
			- 也可以使用自增 ID (autoIncrement - 键生成器)。
			  logseq.order-list-type:: number
		- 索引能力：可以使用数据中的特定属性来创建索引以提高搜索效率，使用索引可以避免遍历所有数据，提升数据检索性能。

- 13:35 什~~~~么是**结构化克隆算法**？ #reading
  id:: 69521317-6d76-4ecb-828f-10b9aa1a6b05
	- 通俗地讲，这是浏览器用来**深度拷贝** JavaScript 对象的一种内部机制。
	- 在 Javascript 中转换 JSON 对象时 (比如 JSON.stringify) 会丢失 Date 对象、正则表达式、Blob 或循环引用等内容，而“结构化克隆算法”可以完整地复制这些类型。
- 14:00 学习 [[Learn IndexedDB]] 中事务的原子性/回滚 #reading
	- IDB 事务具有原子性，也就是“全有或全无”。
	- 比如在事务中有一个请求失败了，该错误会冒泡到事务，最后到达数据库对象。若此时没有调用 `stopPropagation()` 来阻止冒泡，那么错误会冒泡到数据库对象并触发自动回滚。IDB 以此来保证操作的“原子性”。
