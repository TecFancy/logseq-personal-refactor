- ## 📖 Daily Template 使用说明书 (SOP)
  collapsed:: true
	- **适用场景** 每天早上开始工作前 (按需删除 Work 模块)。
	- **核心心法**:
		- **抓大放小**：只记录关键产出，不要记流水账。
		- **清空收件箱**：Inbox 是用来“清空”的，不是用来“堆积”的。
		- **公私分明**：Work 区专注交付，Personal 区专注成长。
	- **🧠 记录位置决策指南 (Q&A)**
		- **Q1: 上班学到的通用技术，要挪到 Personal 吗？**
			- **A: 不需要。** 留在 `Work > TIL`。
			- *理由*: 它是工作的副产品，保留“工作产出”的上下文。通过链接 (``[[Topic]]``) 实现知识复用即可。
		- **Q2: 为了做项目而学的技术，放 Reading 还是 Dev？**
			- **A: 放 `Indie Dev` (或 Coding)。**
			- *原则*: **输出优先于输入**。将学习笔记作为项目任务的“子节点”，体现“为了解决问题而学习”的路径。
			- *例外*: 只有当你暂停开发，专门花整块时间去系统啃一本书/看一套课时，才放 `Reading & Learning`。
	- ---
	- ### 1. 💼 Work 模块详解
		- #### 📥 Inbox & Capture (收集箱)
			- **定义**：任务的**临时停机坪**。
			- **什么东西放这里？**
				- 1. **非会议产生的任务**：同事口头需求、Slack/邮件消息。
				- 2. **突发灵感**：突然想到要重构代码，先丢进来，别打断当前心流。
			- **操作流**：
				- 写入任务 -> 加上 `#SCHEDULED` 或 `#DEADLINE`。
				- **关键动作**：开始做的时候，**拖拽 (Move)** 到 `💻 Coding` 区域，不要留在这里。
		- #### 🤝 Meetings (会议)
			- **08:40 BPML Stand-Up** (已预设 Lite Meeting 结构)：
				- **📝 Discussion (讨论纪要)**：
					- 记录昨日进度、今日计划或遇到的阻碍。
					- *技巧*：按住 `Shift` 点击右侧边栏查看昨日日志，快速总结。
				- **🚀 Action Items (待办)**：
					- 仅记录需要跟进的具体任务。
			- **其他会议**：
				- 使用 `Template/Lite Meeting` (短会/对齐) 或 `Template/Standard Meeting` (正式评审)。
				- 会议产生的 TODO，直接留在会议记录的 `Action Items` 里，**保留上下文**。
		- #### 💻 Coding (核心产出)
			- **定义**：你的**工作台**。
			- **操作流**：
				- 从 Inbox 拖过来的任务，在这里变为 `DOING`。
				- 复杂的实现细节、代码片段、Commit Log 都记录在这个任务的子节点里。
		- #### 💡 Today I Learned (TIL)
			- **定义**：**知识**的入口。
			- **操作流**：
				- 遇到 Bug -> 简记 `#BugFix`。
				- 学到 API -> 简记 `#Snippet`。
				- **切记**：不要写长篇大论，多用链接指向 `[[Concept Page]]`。
	- ---
	- ### 2. 🏠 Personal 模块详解
		- #### 📥 Inbox & Capture
			- 生活杂事（买东西、缴费）或个人灵感。
		- #### 📚 Reading & Learning
			- 记录读书笔记、课程学习。
			- 所有的 **Flashcards (#card)** 建议在此处随上下文产生。
		- #### 🚀 Indie Dev (独立开发)
			- 这是你的“个人实验室”。
			- 用于区分公司的业务代码。在这里记录你的 Side Project、技术探索或造轮子。
		- #### 3. 🗓️ 周末与假期处理
			- **场景**：周六/周日。
			- **动作**：
				- 调用模板后，鼠标点击 `💼 Work` 的圆点。
				- 按下 `Backspace` 或 `Del` **直接删除整个 Work 模块**。
				- 享受生活！
